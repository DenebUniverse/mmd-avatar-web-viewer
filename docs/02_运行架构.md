# 02 运行架构

当前架构：

```text
启动命令 -> scripts/scan_public_assets.py
public/models + public/motions + public/poses -> resource_manifest.json
resource_manifest.json -> 前端角色 / 动作 / 姿态列表
PMX/PMD -> MMDLoader -> THREE.SkinnedMesh
VMD -> MMDLoader.loadAnimation -> AnimationClip
VPD -> MMDLoader.loadVPD -> Pose Data
AnimationClip -> MMDAnimationHelper.add(mesh, { animation })
每帧 -> MMDAnimationHelper.update(delta)
```

核心文件：

```text
src/main.js
src/style.css
index.html
scripts/scan_public_assets.py
public/resource_manifest.json
public/models/avatar_manifest.json
public/motions/vmd_manifest.json
```

## 自动资源索引

`npm run dev` 和 `npm run build` 都会先运行：

```bash
python3 scripts/scan_public_assets.py
```

扫描范围：

```text
public/models/**/*.pmx|*.pmd
public/motions/**/*.vmd
public/poses/**/*.vpd
```

扫描脚本会做基础格式判断：

```text
PMX/PMD：文件头合法
VMD：Vocaloid Motion Data 0002
VPD：Vocaloid Pose Data file
```

然后输出：

```text
public/resource_manifest.json
```

## manifest 合并规则

```text
public/models/avatar_manifest.json  -> 角色显示名、默认角色、附件模型、来源说明
public/motions/vmd_manifest.json    -> VMD/VPD 显示名、来源链接、说明
public/resource_manifest.json       -> 运行时实际读取的合并索引
```

显示逻辑：

```text
1. manifest 中已登记的资源优先使用 manifest 的 id 和 label。
2. manifest 未登记但扫描到格式合法的资源，也会进入前端列表。
3. 扫描资源默认用文件名作为显示名，并标记为“扫描”。
4. 默认角色为 hiying / 绯英。
```

## 角色加载

URL 参数：

```text
?character=hiying
?character=qianyeBlade
?model=/models/local/custom.pmx
```

`?model=` 可绕过角色 ID，直接加载任意公开路径下的 PMX/PMD。

## 动作加载

URL 参数：

```text
?motion=breath
?motion=shotenTime&face=vmd&lip=vmd
?vmd=/motions/local/custom.vmd
```

`?motion=` 使用 manifest / 扫描索引中的 ID。`?vmd=` 可直接加载指定 VMD 路径。
