# 04 动作与表情系统

## 动作系统

动作列表来自 `public/resource_manifest.json`。服务启动前会扫描 `public/motions/**/*.vmd`，manifest 中声明的资源使用 manifest 显示名，未声明但格式合法的 VMD 也会进入动作下拉框。

常用快捷按钮对应这些 VMD/VPD 资源：

```text
breath     -> breath_hiying_compatible.vmd
idle       -> builtin_idle.vmd
wave       -> builtin_wave.vmd
jump       -> builtin_jump.vmd
peek       -> builtin_peek.vmd
shotenTime -> shoten_time_mihoyo_yujie.vmd
goodTea    -> good_tea_shake_mihoyo_yujie.vmd
lovermaxHipSway -> lovermax_tiktok_hip_sway_mihoyo_compatible.vmd
pose       -> default_stand.vpd
```

## 角色适配

自动检查会读取 PMX 的 bone/morph 表，再读取 VMD 的 bone/morph 关键帧名称，逐项比对：

```bash
python3 scripts/check_character_compatibility.py
```

当前结果：绯英与千冶·刃均通过 `breath / idle / wave / jump / peek / shotenTime / goodTea / lovermaxHipSway` 的名称级兼容检查。该检查能验证 VMD 能按名称驱动 PMX，但不等于肉眼完全无穿模；长动作仍需要浏览器或 MMD 预览确认衣服、袖子、头发和脚底表现。

## 表情层

两个角色都包含常用 morph：

```text
笑い / 喜び / 怒り / 困る / まばたき / あ / い / う / え / お
```

前端表情按钮会写入 `笑い / 喜び / 怒り / 困る`。播放含 face morph 的 VMD 时，建议使用：

```text
?face=vmd
```

## 口型层

前端 TTS/麦克风会写入：

```text
あ / い / う / え / お
```

播放含 lip morph 的 VMD 时，建议使用：

```text
?lip=vmd
```

## 原始呼吸包

用户上传的 `呼吸モーション` 包已保留在：

```text
public/motions/external/breath_original/
```

该包依赖追加骨骼。为保证当前两个角色默认可见效果，项目同时提供了使用标准骨骼的兼容呼吸 VMD：

```text
public/motions/generated/breath_hiying_compatible.vmd
```

## lovermax 动作

`lovermaxHipSway` 播放 `/motions/external/lovermax_tiktok_hip_sway_mihoyo_compatible.vmd`。原始 VMD 依赖部分绯英/千冶·刃不存在的指0、涙、舌和部分表情 morph，因此项目注册的是名称兼容版；原始文件和 README 仍保留在 `public/motions/external/` 供查验。
