# 05 自动检查与验收标准

运行：

```bash
npm run check
```

检查内容：

- 自动扫描 `public/models`、`public/motions`、`public/poses` 并生成 `public/resource_manifest.json`。
- 验证新放入的合法 VMD 可以不改配置进入动态 manifest。
- 绯英 PMX 模型和贴图存在。
- 千冶·刃 PMX 模型和贴图存在。
- 前端使用 `MMDLoader`、`MMDAnimationHelper`、`loadAnimation`、`loadVPD`。
- VMD/VPD 资源头部可解析。
- 昇天time、好茶摇一摇、lovermax 动作已进入 `public/motions/external/`。
- DOM 绑定和控制面板元素完整，角色 / VMD / VPD 下拉框存在。
- 角色适配检查通过：绯英与千冶·刃均具备当前关键 VMD 动作所需 bone/morph 名称。
- dist fallback 与 public 资源同步。
- 前端 smoke test 能模拟加载默认呼吸 VMD，并确认角色、动作、姿态列表可用。

人工验收：

```text
[ ] npm run dev 能打开页面
[ ] 默认加载绯英 PMX
[ ] 角色下拉框可切换到千冶·刃
[ ] 新放入 public/models 的有效 PMX/PMD 重启后出现在角色列表
[ ] 新放入 public/motions 的有效 VMD 重启后出现在动作列表
[ ] 新放入 public/poses 的有效 VPD 重启后出现在姿态列表
[ ] 切换角色后当前动作继续播放或重新加载成功
[ ] 默认呼吸动作可见
[ ] 昇天time 按钮可播放
[ ] 好茶摇一摇按钮可播放
[ ] lovermax 腰振り按钮可播放
[ ] 半身/全身切换正常
[ ] OBS ?obs=1 不显示控制台
[ ] VMD 表情/口型模式不会被前端按钮误覆盖
```
